## How to Contribute

  <Step>
    
  ### 1. Reporting Issues
    
  If you encounter any problems or have questions, please open an issue on our [GitHub repository](https://github.com/your-repo/issues).

  <Alert type='warning' title='Check Existing Issues'>
    Before creating a new issue, please search to see if it has already been reported or addressed.

  ### 2. Suggesting Enhancements
    
  We're always looking for ways to improve. If you have ideas for enhancements:

{' '}

<ul>
  <li>Open an issue with the tag `enhancement`.</li>
  <li>Provide as much detail as possible.</li>
  <li>Discuss your idea with the community.</li>
</ul>

  </Step>

  <Step>

### 3. Submitting Pull Requests

To submit changes, follow these steps:

1. **Fork the Repository**: Click the **Fork** button at the top right corner of the repository page.

2. **Clone Your Fork**:

   ```bash
   git clone https://github.com/your-username/your-repo.git
   ```

3. **Create a Branch**:

   ```bash
   cd your-repo
   git checkout -b feature/your-feature-name
   ```

4. **Make Changes**: Edit the `.mdx` files using FumaDocs components. Ensure consistency with existing documentation.

5. **Commit Changes**:

   ```bash
   git add .
   git commit -m "Add your feature or fix"
   ```

6. **Push to Your Fork**:

   ```bash
   git push origin feature/your-feature-name
   ```

7. **Open a Pull Request**: Go to your fork on GitHub and click on **Compare & pull request**. Fill out the pull request template and submit.

  <Alert type="success" title="Thank You!">
    We appreciate your effort in making our documentation better!
  </Alert>
  
  </Step>
</Steps>

## Style Guidelines

To maintain consistency, please follow these guidelines:

- **Writing Style**

  - Use clear, concise language.
  - Write in the active voice.
  - Use American English spelling.

- **Formatting**

  - Use Markdown for content and MDX for embedding components.
  - Utilize FumaDocs components like `<Alert>`, `<Steps>`, and `<Step>`.

- **Code Blocks**

  - Use backticks for code snippets.
  - Specify the language for syntax highlighting.

## Development Setup

To set up a local development environment:

1. **Install Dependencies**:

   ```bash
   npm install
   ```

2. **Start the Development Server**:

   ```bash
   npm run dev
   ```

3. **Open in Browser**: Navigate to [http://localhost:3000](http://localhost:3000) to view the documentation.

<Alert type='info' title='Hot Reloading'>
  Changes you make to the files will automatically reload the page.
</Alert>
